# SNAP Emergency Assistance Program

The SNAP Emergency Assistance Program provides crucial support to participants facing food insecurity during their journey through the Cross Purpose program.

## Overview

This program offers:
- Emergency SNAP application assistance
- Support for participants in Job Readiness and Supervised Job Search phases
- Verification and tracking of SNAP status
- Coordination with coaches and participants

## Eligibility

Participants are eligible when they are in:
- **Job Readiness Phase**: Actively preparing for employment
- **Supervised Job Search Phase**: Actively seeking employment

## Program Flow

### 1. Participant Request
Participants can request SNAP assistance through:
- The Explorer dashboard
- Direct communication with their coach
- The participant portal's assistance section

### 2. Verification Process
The system verifies:
- Current participant phase (Job Readiness or Supervised Job Search)
- SNAP application status
- Previous assistance requests

### 3. Support Delivery
Once approved:
- Participant receives guidance on SNAP application
- Coach is notified of the request
- Status is tracked in Salesforce
- Follow-up notifications are sent

## For Coaches

<AccordionGroup>
  <Accordion title="Viewing SNAP Requests">
    Coaches can view all SNAP assistance requests in their dashboard under the "Participant Support" section.
    
    Each request shows:
    - Participant name and current phase
    - Request date
    - Current status
    - Any notes or updates
  </Accordion>

  <Accordion title="Processing Requests">
    To process a SNAP assistance request:
    
    1. Review the participant's eligibility
    2. Verify their current phase
    3. Contact the participant to discuss their situation
    4. Guide them through the SNAP application process
    5. Update the request status in the system
  </Accordion>

  <Accordion title="Tracking Completion">
    Mark requests as complete when:
    - The participant has successfully applied for SNAP
    - Documentation is received
    - The request is synced with Salesforce
  </Accordion>
</AccordionGroup>

## Technical Implementation

### Database Schema

```sql
model SNAPAssistanceCompletion {
  id            String   @id @default(cuid())
  participantId String
  participant   User     @relation(fields: [participantId], references: [id], onDelete: Cascade)
  completedAt   DateTime @default(now())
  salesforceId  String?  // Supportive Service record ID
  notes         String?
  
  @@index([participantId])
  @@index([completedAt])
}
```

### Key API Endpoints

<CodeGroup>
```typescript GET /api/snap-assistance/status
// Check if participant is eligible and hasn't already completed
GET /api/snap-assistance/status?participantId={id}

Response:
{
  "eligible": boolean,
  "currentPhase": string,
  "hasCompleted": boolean,
  "completedAt": string | null
}
```

```typescript POST /api/snap-assistance/complete
// Mark SNAP assistance as completed
POST /api/snap-assistance/complete

Body:
{
  "participantId": string,
  "notes": string
}

Response:
{
  "success": boolean,
  "salesforceId": string,
  "completedAt": string
}
```
</CodeGroup>

## Salesforce Integration

SNAP assistance completions are automatically synced to Salesforce as **Supportive Service** records with:

- **Service Type**: "SNAP Application Assistance"
- **Status**: "Completed"
- **Related Contact**: Participant's Salesforce Contact ID
- **Completion Date**: Timestamp of completion

## Email Notifications

Automated emails are sent for:

1. **Participant Confirmation**: When SNAP assistance is marked complete
2. **Coach Notification**: When a participant requests assistance
3. **Admin Alert**: For system errors or failed syncs
